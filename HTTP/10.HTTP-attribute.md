# 1. HTTP 메서드의 속성

## 1.1. 안전
- Safe
- 호출해도 리소스를 변경하지 않는다.

## 1.2. 멱등
- `Idempotent`
- 한 번 호출하든 두 번 호출하던 100번 호출하든 결과가 똑같다.

### 1.2.1. 멱등 메서드
- GET
- PUT
- DELETE

### 1.2.2. 멱등 활용
- 자동 복구 메커니즘


## 1.3. 캐시 가능
- GET, HEAD, PATCH, POST 캐시로 가능
- GET, HEAD 정도만 캐시로 사용
- POST, PATCH는 본문 내용까지 `캐시 키로 고려`해야 하는데 구현이 쉽지 않음
